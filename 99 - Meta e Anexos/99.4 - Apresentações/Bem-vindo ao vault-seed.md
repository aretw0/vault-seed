# Bem-vindo ao vault-seed

O vault-seed é um template Git que transforma notas Markdown em um sistema completo: vault local, site publicado, Lab de análise e publicação multi-canal — no mesmo repositório, sem plataformas proprietárias.

Esta seção apresenta o projeto progressivamente. Siga a ordem das páginas na barra lateral para construir o entendimento passo a passo.

## Os quatro pilares

| Pilar | O que é | Como acessar |
|---|---|---|
| **Vault** | Suas notas em Markdown, organizadas com o método PARA | Editor local: Obsidian, VS Code ou qualquer editor |
| **Site** | O vault publicado como site estático com Starlight e Astro | GitHub Pages — automático a cada push |
| **Lab** | Notebooks Marimo versionados para análise e apresentações | Seção Lab neste site |
| **Outbox** | Fila curada de publicação para RSS, canais e feeds externos | Ver [[Outbox Soberana de Publicação]] |

## O que você encontra aqui

O site que você está visitando é a demonstração do próprio template. As notas publicadas mostram o que você recebe ao inicializar um vault a partir deste template — incluindo guias, referências e os notebooks do Lab.

As seções na barra lateral seguem intenções de uso:

- **Começar** — este índice e a visão geral do projeto
- **Organizar** — métodos PKM, PARA, links, tags e MOCs
- **Explorar** — Lab, dados, diagramas e a página de exploração interativa
- **Publicar** — site, RSS, fluxo de publicação e outbox
- **Automatizar** — GitHub Actions, CI, agentes de IA e pipelines
- **Manter** — qualidade, curadoria editorial e versionamento
